

    MongoDB介于关系型数据库和非关系型数据库之间

    MongoDB和MySQL对比：

            MySQL      |      MongoDB     |     解释/说明
        ————————————————————————————————————————————————————
            database   |      database    |   数据库-数据库
        ————————————————————————————————————————————————————
            table      |     collection   |      表-集合
        ————————————————————————————————————————————————————
             row       |      document    |    记录/行-文档
        ————————————————————————————————————————————————————
            column     |       field      |      字段-域
        ————————————————————————————————————————————————————
            index      |       index      |     索引-索引
        ————————————————————————————————————————————————————
            joins      |                  | 表连接-MongoDB不支持
        ————————————————————————————————————————————————————
          primary key  |    primary key   | 主键-MongoDB自动将_id设为主键

    MongoDB三元素：
        文档、集合、数据库

        1、文档（document）
            {'name':'asc', 'age':'15'}

        2、集合（collection）：多个文档，相当于MySQL中的表
            {'name':'asc', 'age':'15'}
            {'name':'asc', 'grade':'100'}
            {'title':'xxx', 'price':'30'}


    MongoDB基本命令：

        db：查看当前数据库

        show dbs：查看所有的数据库

        dropdatabase：删除数据库

        use 数据库名：使用某个数据库，若数据库不存在也可以用此方法创建数据库
                    【Tip】使用此命令不能完全创建，会保存在内存中，
                           插入数据后才能完全创建

        db.集合名.insert(value)：添加数据到指定集合

        db.集合名.find()：从指定集合中查找数据
